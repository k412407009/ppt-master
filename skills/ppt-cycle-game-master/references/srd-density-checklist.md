# SRD Density Checklist — 单循环节奏密度量化清单

> **SRD = Single-Run Density**（单循环节奏密度）
> 一份 7 档时间锚点的量化模板，用来在 design_spec.md §V 强制 AI 把"玩家头 30 秒到 D30"的钩子分布写清楚。
>
> 本文件**必读**于 Step 4 编写 design_spec §V 前。
> 写完 SRD 表后，AI **必须** 跑一遍本文件 §3 的反模式自检。

---

## §1 SRD 7 档锚点定义

| 阶段 | 时长 | 玩家心智状态 | 必触钩子类型 | 留存目标 |
|---|---|---|---|---|
| **5.1** | **0–30s** | 第一印象 | 视觉钩子 ×1（首屏 hero） | **95%** |
| **5.2** | 30s–2min | 摸索 | 反馈循环 ①（点击 → 即时反馈） | **80%** |
| **5.3** | 2–15min | 试玩主循环 | 核心玩法首演（一个完整 loop 走完） | **70%** |
| **5.4** | 15–60min | 加深熟练 | 第二机制引入（解锁新功能） | **60%** |
| **5.5** | 1–4h | 系统耦合 | 元层进阶（多系统互锁） | **50%** |
| **5.6** | **Day 2–3** | 复访验证 | **战斗 / 反派 / 第一次高峰** | **40%** |
| **5.7** | Day 5–30 | 长期粘性 | LTV / 社交 / 赛季 / 长期目标 | **20%** |

| 字段 | 必填 | 说明 | 示例 |

| 阶段 | ✅ | 5.1 ~ 5.7 七档 | `5.1` |

| `后期` | `5.6-5.7 (Day 2+)` |
| `很快` | `≤2 秒` |
| `稍后` | 具体时间锚点 |

## §3 反模式自检（写完后必跑）

### 3.1 起点反模式

```
❌ 5.1 起步在"30 分钟"
   → 头 30min 没钩子，流失 48-50%（行业平均）
✅ 5.1 起步必须 ≤ 30s（视觉钩子在首屏）
```

### 3.2 断点反模式

```
❌ 5.4 (30min-1h) → 5.5 (1-3h) → 5.6 (Day 3-Day 7)
   断点：Day 1 末 - Day 2 - Day 3 之间无钩子描述

✅ 5.4 (15-60min) → 5.5 (1-4h) → 5.6 (Day 2-3) → 5.7 (Day 5-30)
   连续：每档时长无缝衔接，Day 2-3 必有新机制
```

### 3.3 战斗/反派延迟反模式

```
❌ 战斗 / 反派 / PvE 在 Day 7 才出
   → 玩家不知道游戏有"战斗"概念，第一周流失加速

✅ 战斗 / 反派 / PvE 必须在 5.6 (Day 2-3) 内出现
   → 给玩家"游戏会变化"的预期，留存 +5-8%
```

### 3.4 单位歧义反模式

```
❌ "思考头 30，挂机回报 45"
   → 用户/评委必问 "30 秒还是 30 分钟？"

✅ "思考头 30s，挂机回报 45s"
   → 单位明确，不再被 challenge
```

### 3.5 留存目标反模式

```
❌ 留存目标全部填"高"或"中"或"低"
   → 不可量化，评审会被 challenge

✅ 留存目标必须填具体百分数（5.1=95% 5.7=20%）
   → 行业可对标，评审可信
```

## §4 SRD 与游戏类型的适配

不同题材的 SRD 重点不同。以下是本 skill 支持的 3 种题材的差异：

### 4.1 单机循环经营（如 MPH / Whiteout / Animal Shelter）

| 阶段 | 重点 |
|---|---|
| 5.1 | 第一个雇员上岗 / 第一只动物入园 |
| 5.2 | 第一笔收入入账 / 第一次升级 |
| 5.3 | 完整一轮"接客→收入→升级"循环 |
| 5.4 | 解锁第二种 NPC / 第二种动物 / 第二个店面 |
| 5.5 | 离线收益启动 / 元层货币上线 |
| 5.6 | 第一次"高峰事件"（圣诞活动 / 突发任务 / 反派出现） |
| 5.7 | 赛季 / 排行榜 / 长期目标系统 |

### 4.2 竖屏塔防（如 Thronefall / Kingdom Rush 竖屏化）

| 阶段 | 重点 |
|---|---|
| 5.1 | 看到首关地图 + 第一个塔位置 |
| 5.2 | 第一波 1 只小怪 → 第一个塔击杀 |
| 5.3 | 完整一关（5 波内）通关 |
| 5.4 | 解锁第二种塔 / 第二种兵种 |
| 5.5 | 升级树 / 元层货币 / 装备系统 |
| 5.6 | Boss 战 / 双线压制 / 第一个失败关卡 |
| 5.7 | 无尽模式 / 赛季 / 排行榜 |

### 4.3 工厂链 / 产业化（如 Mindustry / Factorio 简化版）

| 阶段 | 重点 |
|---|---|
| 5.1 | 看到首屏：一个采集点 + 一个工厂 |
| 5.2 | 第一条传送带连通 → 第一件产品产出 |
| 5.3 | 完整一条 3-节产业链（采集 → 加工 → 销售） |
| 5.4 | 解锁第二种资源 / 第二种产品 |
| 5.5 | 自动化升级 / 离线产出 / 元层科技树 |
| 5.6 | 第一次"产业危机"（资源枯竭 / 罢工 / 竞争对手） |
| 5.7 | 行业排行 / 赛季 / 跨服竞争 |

## §5 SRD 表的"密度"概念

"密度"指的是**每个时间窗口里钩子的数量**：

| 阶段 | 钩子数 | 推荐密度 |
|---|---|---|
| 5.1 | 1 | 1 个超强钩子（视觉钩子） |
| 5.2 | 2 | 2 个互相关联的钩子（操作 + 反馈） |
| 5.3 | 3 | 3 个递进钩子（学 → 用 → 升级） |
| 5.4 | 3-5 | 3-5 个钩子（解锁 + 自定义 + 收集） |
| 5.5 | 4-6 | 4-6 个钩子（系统耦合 + 元层 + 离线） |
| 5.6 | 必有 1 个 BIG | 1 个高峰事件 + 几个常规 |
| 5.7 | 长期钩子 | 1-2 个长期 + N 个赛季性 |

**密度低 = 流失风险高**：
- 5.1 钩子数 = 0 → 5s 内滑掉
- 5.3 钩子数 = 1 → 玩家觉得"就这？"
- 5.6 没有 BIG → D2 不会回访

## §6 SRD 表的"反例库"（来自真实复盘）

### 反例 1：动物联盟早期版（已修复）

```
原版 SRD（被用户 challenge）：
| 阶段 | 时长 |
|---|---|
| 5.4 | 30 分钟到 1 小时 |
| 5.5 | 1 到 3 小时 |
| 5.6 | Day 3 到 Day 7  ← Day 2 在哪？

修复后：
| 阶段 | 时长 |
|---|---|
| 5.4 | 15-60min |
| 5.5 | 1-4h |
| 5.6 | Day 2-3 |  ← Day 1 末塔防 + Day 2 早潜入
| 5.7 | Day 5-30 |
```

### 反例 2：单位歧义

```
原版：思考头 30，挂机回报 45
用户 challenge："30 秒还是 30 分钟？"

修复：思考头 30s，挂机回报 45s（首循环），1.5min（中期），3min（末期）
```

## §7 SRD 表的"评审说服力"

写好 SRD 表的好处（评审视角）：
1. **节奏可量化** → 评委可以直接 cue "5.6 Day 2-3 战斗设计具体细节"
2. **留存可对标** → 行业 D1=40-60%, D7=15-25%, D30=5-12%
3. **断点暴露 = 风险暴露** → 写不出 5.5 → 5.6 = 这部分还没想清楚
4. **七档 = 评审导航** → 评委可以挑感兴趣的档单独问

写不好 SRD 表的代价：
- 评委问"D2 玩家在干什么"答不上来 → 立项被打回
- 评委问"头 30 秒玩家看到什么"答"一会儿就玩起来了" → 立项被打回
- SRD 表只有 5.1-5.5（缺 5.6/5.7）→ "这游戏只想做短线"印象，被打回
