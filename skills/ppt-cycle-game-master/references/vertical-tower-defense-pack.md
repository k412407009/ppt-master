# Vertical Tower-Defense + Industrialization Pack — 竖屏塔防 + 产业化专题

> 适用题材：竖屏塔防 / 工厂链 / 产业化经营 / 资源链管理。
>
> 核心特点：**单局短** + **重数值** + **多波次密度** + **上中下游产业链**。
> 与单循环 pack 的差异：5.6 必有 BOSS、必有数值升级树、必含资源链。

## §3 SRD 表特化（塔防 + 工厂视角）

通用 SRD 见 `${SKILL_DIR}/references/srd-density-checklist.md`，本 pack 给塔防/工厂特化的钩子建议：

### 3.1 竖屏塔防 SRD

| 阶段 | 钩子建议 |
|---|---|
| **5.1 (0-30s)** | 看到首关地图 + 自动出现的第一个塔（无需玩家操作） |
| **5.2 (30s-2min)** | 第一波 1 只小怪 → 第一个塔自动击杀 + 升级提示 |
| **5.3 (2-15min)** | 完整一关（5-8 波） → 通关结算 + 1 颗星 |
| **5.4 (15-60min)** | 解锁第二种塔 / 第二种兵种 + 第一次手动放塔 |
| **5.5 (1-4h)** | 升级树 / 元层货币 / 装备系统 + 第一次"调配兵种"决策 |
| **5.6 (Day 2-3)** | **首个 Boss 战** + 第一次失败关卡（不让玩家觉得"游戏太简单"） |
| **5.7 (Day 5-30)** | 无尽模式 + 赛季 + 公会 + 排行榜 |

### 3.2 工厂链 SRD

| 阶段 | 钩子建议 |
|---|---|
| **5.1 (0-30s)** | 看到首屏：1 个采集点 + 1 个工厂（已自动连接） |
| **5.2 (30s-2min)** | 第一条传送带连通 → 第一件产品产出（数字跳动） |
| **5.3 (2-15min)** | 完整 3 节产业链（采集 → 加工 → 销售）+ 第一笔利润 |
| **5.4 (15-60min)** | 解锁第二种资源 → 强制玩家做第二条链 |
| **5.5 (1-4h)** | 自动化升级（传送带升级 / 机器人） + 离线产出 |
| **5.6 (Day 2-3)** | **首个产业危机**：资源枯竭 / 罢工 / 竞争对手出现 |
| **5.7 (Day 5-30)** | 行业排行 + 跨厂区扩张 + 赛季内容 |

## §6 数值树与升级路径（塔防/工厂特有）

塔防/工厂游戏立项 PPT **必含**数值树章节（4-5 页）：

### 6.1 必出图表

| 图表 | 内容 |
|---|---|
| **DPS 升级曲线** | 每级塔/兵种的 DPS 增长（线图） |
| **资源消耗曲线** | 每级升级的资源消耗（线图，对比 DPS 收益） |
| **兵种树** | 3 维度 × 5-7 级 的全展开树（节点图） |
| **波次密度图** | 5.1-5.7 各档"敌人波次/分钟"密度（柱图） |
| **资源链路图** | 上中下游产业链（节点+流向） |

### 6.2 数值平衡说明

每个升级级别必填：
- DPS / 资源消耗
- TTK（Time To Kill）
- 性价比（DPS / 消耗）
- 解锁条件（玩家行为锚点）

## §7 产业化机制（工厂/产业经营特有）

### 7.1 上中下游模型

```
上游（采集）→ 中游（加工）→ 下游（销售）
   矿石         零件          成品
   木材         板材          家具
   农作物       食材          餐厅菜品
```

### 7.2 必出 KPI

| 指标 | 说明 |
|---|---|
| 单链利润率 | 一条 3 节链的总收入/总成本 |
| 链路效率 | 单位时间产出 / 物流损耗 |
| 自动化率 | 玩家手动操作占比（应随时间下降） |
| 复杂度阈值 | 超过 N 个工厂时玩家弃游率 |

### 7.3 反模式

```
❌ 上中下游 = 3 选 1（不是产业化，是单链）
   产业化必须 ≥3 条平行链，且链与链之间有依赖

✅ 上中下游 = 3 条平行链 + 跨链依赖
   （链 1 的下游产品 = 链 2 的上游原料）
```

## §8 竖屏 vs 横屏 选择

| 场景 | 选竖屏 | 选横屏 |
|---|---|---|
| 单手操作 | ✅ | ❌ |
| 信息密度高（>6 资源同屏） | ❌（屏幕窄） | ✅ |
| 单关时长 ≤5min | ✅ | 也行 |
| 单关时长 ≥15min | ❌（玩家坐不住） | ✅ |
| 移动端为主 | ✅ | 也行 |
| Steam / 主机为主 | ❌ | ✅ |

### 8.1 PPT 画布选择

| 场景 | PPT 画布 | 评审会投影 |
|---|---|---|
| 项目最终是竖屏 | `ppt169` (评审用) + design_spec §VIII 含 9:16 截图 | ✅ 横屏投影 + 9:16 截图嵌入 |
| 项目最终是横屏 | `ppt169` | ✅ 横屏投影 |
| 项目同时支持横竖屏 | `ppt169` + 双格式截图 | ✅ |

> ⚠️ **不建议**用 `xhs` (1080×1920) 画布做评审 PPT —— 评审会都是横屏投影，竖屏 PPT 会浪费 2/3 屏幕。
