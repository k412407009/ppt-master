# Cycle-Game Pitch Pack — 单机循环游戏立项专题

> 适用题材：MPH-style / Animal Shelter / Whiteout Survival / Cat Cafe / 任何 "进入 - 操作 - 反馈 - 升级 - 退出" 的单循环游戏。
>
> 核心特点：**不会有太多融合与过渡**——单核心循环 + 元层进阶。
> 故 **不需要** 双调色板、双玩法切换、Pixar 角色情感弧。

---

## §1 题材判定

### 1.1 适用本 pack ✅

| 子类型 | 代表作 | 核心循环 |
|---|---|---|
| 经营管理 | MPH, Cat Cafe, Animal Shelter | 接客→收入→升级 |
| 放置挂机 | Whiteout Survival, AFK Arena | 离线收益→上线消耗→升级 |
| 模拟生存 | Project Zomboid（简化版） | 探索→采集→建造→生存 |
| 卡牌收集（轻度） | Storage Auctions | 抽卡→收集→展示 |
| 城市建造（轻度） | Pocket City | 规划→建造→收税 |

### 1.2 不适用 ❌（用 ppt-master 主路径）

| 子类型 | 原因 | 改用 |
|---|---|---|
| 多线叙事（双调色板/双玩法） | 本 pack 故意不支持 | `ppt-master` + `executor-consultant-top.md` |
| 重剧情 RPG | 需要角色情感弧 | `ppt-master` + 自由设计 |
| MMO / PvP | 不是"单循环" | `ppt-master` + 自由设计 |
| Roguelike（含 procedural） | 单循环但含随机性，需要专项 SRD | `ppt-master` + 自定义 |

## §2 必参考竞品（按维度分）

立项 PPT 必须按下表 4-5 个维度选 3-5 款竞品。

| 维度 | 选什么 | 抽帧重点 |
|---|---|---|
| **核心循环** | 与本游戏循环最像的 1 款 | 完整循环 5-8 步截图 |
| **美术风格** | 风格最接近的 1 款 | 美术 hero 图 + UI 大图 |
| **商业化** | 商业化最成功的 1 款 | 商城页 + 礼包 + IAP 弹窗 |
| **元层进阶** | 元层做得最好的 1 款 | 升级树 + 赛季页 + 公会页 |
| **UI / UX**（可选） | UI 最干净的 1 款 | 主界面 + 菜单 + Tooltip |

### 2.1 推荐组合（按子类型）

#### 经营管理类
```
核心循环：MPH                  （5-15min 一局，循环明确）
美术风格：Cat Cafe Manager     （温暖卡通）
商业化：  MPH                  （IAP+广告组合拳）
元层：    Animal Shelter       （收集 + 升级 + 解锁）
UI：      Pocket City          （干净 + 信息密度低）
```

#### 放置挂机类
```
核心循环：Whiteout Survival
美术风格：AFK Arena 或 Idle Heroes
商业化：  Whiteout Survival
元层：    Whiteout Survival（科技树 + 联盟）
UI：      AFK Arena
```

#### 卡牌收集类（轻度）
```
核心循环：Storage Auctions
美术风格：TCG Card Shop Simulator
商业化：  Storage Auctions
元层：    收集图鉴系统
UI：      自由
```

## §3 SRD 表特化（单循环视角）

> 通用 SRD 表见 `${SKILL_DIR}/references/srd-density-checklist.md`。
> 本 pack 给"单循环"特化的钩子建议：

| 阶段 | 单循环游戏的钩子建议 |
|---|---|
| **5.1 (0-30s)** | 主角立绘 + 一句温暖语音（"欢迎来到我的咖啡馆！"）+ 1 个 Tap |
| **5.2 (30s-2min)** | 第一笔小额收入 / 第一只动物入园 / 第一位顾客上门（数字跳动 + 音效） |
| **5.3 (2-15min)** | 完整循环走完（接客 → 服务 → 收钱 → 升级 1 次） |
| **5.4 (15-60min)** | 解锁 NPC 2 / 商品 2 / 房间 2（"哦，还有更多东西！"） |
| **5.5 (1-4h)** | 离线收益弹窗 + 元层货币（"哎，我离开 30 分钟，赚了 X！"） |
| **5.6 (Day 2-3)** | **首次"小高峰"事件**：节日/突发任务/罕见客人/Boss 客人 |
| **5.7 (Day 5-30)** | 赛季 / 排行榜 / 收集图鉴完成度 / 联盟 |

### 3.1 与多玩法/塔防的区别

| 对比项 | 单循环游戏 | 双玩法/塔防 |
|---|---|---|
| 5.6 高峰类型 | "节日活动 / 突发客人"（**叙事型**） | "Boss 战 / 双线压制"（**战斗型**） |
| 5.5 元层 | 离线收益 + 元层货币（**佛系**） | 升级树 + 装备（**重数值**） |
| 战斗在 SRD | **不应该有**（单循环游戏不是战斗游戏） | 5.6 必有 |
| 反派 | **可有可无**（动物联盟有偷猎者，但不是必须） | 必有 |

## §6 反模式（单循环游戏特有）

### 6.1 把单循环写成多玩法

```
❌ "我们做单循环管理，但中后期加塔防 + Roguelike + 卡牌"
   → 题材不聚焦，本 pack 不适用，应改用 ppt-master 自由设计

✅ "我们做单循环管理，元层只加'图鉴 + 赛季'，不加新玩法"
```

### 6.2 5.6 用"战斗"作为高峰

```
❌ 单循环管理游戏 5.6 写"BOSS 战"
   → 玩家心智不预期战斗，强行加战斗会破坏循环

✅ 单循环管理游戏 5.6 写"节日活动 / 罕见客人 / 突发任务"
```

### 6.3 双调色板

```
❌ 单循环游戏做 DAY/NIGHT 双调色板（除非游戏本身昼夜变化是核心机制）
   → 双调色板 = 双套素材成本翻倍，单循环游戏不值

✅ 单调色板 + 偶尔氛围切换（如夜晚屏幕暗 10%）
```

## §8 速通模式（单循环游戏特有 fast-forward）

单循环游戏立项 PPT 比多玩法立项简单 30%。
速通触发后预期用时：

| 题材 | 速通用时（参考 I_动物联盟） |
|---|---|
| 单循环（本 pack） | 60-90 分钟 |
| 双玩法融合（如 I_动物联盟） | 90-150 分钟 |
| 塔防 + 产业化 | 90-120 分钟 |
| 多线叙事 + 双调色板 | 150-240 分钟 |

> 用户说"剩余的直接跑完"后，AI 按上表预估给出"~XX 分钟后给报告"。
